----------------- Работа с программой -----------------------

В. Я много работаю с текстовыми файлами, но очень часто эти текстовые файлы имеют совершенно
   непонятные расширения: *.1, *.textfile, *.someext и другие. И каждый раз щёлкать по одному 
   и тому же редактору для каждого редактора уже надоело. Как сделать так, чтобы все новые 
   расширения принудительно открывались в конкретном редакторе?
О. Для этого в OpenFileTC откройте настройки выбранного вами редактора и поставьте галочку
   "Открывать все новые расширения сразу в этом редакторе"


----------------- Настройка программы -----------------------

В. Я хочу отредактировать исполняемый файл, а программа постоянно спрашивает меня -
   хочу ли я добавить этот файл в список редакторов. Как отключить это сообщение?
   И вообще, зачем оно нужно?
О. Эта опция была создана для удобства пользователей. Теперь вы можете добавлять 
   редакторы не только через саму программу OpenFileTC (пункт "Другая программа..."),
   но и через Total Commander. Вам нужно просто нажать на редакторе клавишу F4, и 
   программа (при вашем согласии) добавит этот редактор в список доступных редакторов.
   Для того, чтобы отключить эту опцию, откройте настройки программы, там - раздел
   "Управление", и в этом разделе снимите галочку с опции 
   "F4 на EXE - добавлять EXE в список редакторов"

В. Как принудительно открыть Open File shell for TC? 
О. В настройках OpenFileTC, в разделе Управление, вы можете задать удобный для вас
   способ принудительного запуска программы

В. Как открыть сразу несколько файлов в нужном редакторе? 
О. Для этого нужно выделить эти файлы и нажать F4. На редактирование отправятся все 
   выделенные файлы. 

В. Какие переменные окружения понимает Open File shell for TC? 
О. В файле SupportedTags_rus.txt описаны все тэги, которые поддерживает программа.

В. Как указать для приложения, что оно умеет открывать файлы без расширения? 
О. Этому приложению к списку расширений следует добавить расширение %nil% (5 символов) 

В. Как сделать так, чтобы подсказка появлялась рядом с мышью, а не в углу окна Open File TC? 
О. Для этого нужно в настройках программы поставить галочку в переключателе Показывать 
   подсказки рядом с мышью (раздел Вид) 

В. Что это за новый пункт  в настроках - "Запустить в отладочном режиме"
О. Этот пункт доступен только в том случае, если загружен русский язык для программы. Данная опция
   предлагает пользователям принять участие в разработке и улучшении Open File shell for TC. В случае, 
   если данная опция включена, программа создаст подпапку Debug, в которой будут создаваться файлы с 
   некоторой полезной для разработчиков информацией. Вам как пользователям она может быть непонятна.
   Чтобы принять участие в разработке и улучшении Open File shell for TC, вам нужно все создаваемые файлы 
   упаковывать и отсылать по адресу oftc@antoon-inc.com. Эта акция абсолютно добровольна!

В. В настройках программ стало доступным поле - "Командная строка для всех расширений". Что
   это за поле, и зачем оно мне нужно?
О. Это поле позволяет избавиться от необходимости указывать для каждого нового расширения 
   командную строку. Некоторым программам для открытия файла требуется передать какие-то параметры 
   в командной строке. К примеру, файловый менеджер FAR открывает файлы, если ему указать строку вида: 
   FAR.exe /e C:\text.txt 
   Здесь FAR.exe - это сам файловый менеджер FAR
         /e      - параметр для программы, указывающий, что файл, переданный далее, нужно будет
	           редактировать
	 C:\text.txt - полный путь до файла для редактирования
   В обычном случае это поле можно оставлять пустым - оно требуется только в особых случаях.

----------------- Решение проблем -----------------------
 
В. Стал замечать, что при работе с программой иногда теряется буфер обмена. Это баг?
О. К сожалению, это действительно так. Дело в том, что Open File TC получает список файлов через
   буфер обмена, если запускается из-под TC версий 6.5x и младше.
   Решением является обновление Total Commander до версии 7.0 и выше. В этих версиях Open File TC
   получает список выделенных вами файлов минуя буфер обмена - используя внутренние Windows-команды.

В. Приложение отказывается открывать файл - пишет Не могу открыть файл. 
О. Возможно, этому приложению следует отправлять имена файлов без кавычек. Поэкспериментируйте
   со способом запуска программы, попробуйте выбрать, например, второй пункт. 
 
В. Как разделить программу между пользователями? Сделать так, чтобы каждый пользователь 
   имел свои настройки для OpenFileTC. 
О. Используйте переключатель в окне настроек программы, в разделе Управление. 

В. При запуске какой-то программы через Open File TC программа запускается, но при этом Open File TC 
   остаётся висеть в списке процессов. Это баг? 
О. Нет, это одно из требований к программе: OpenFileTC следит за открытым файлом, чтобы после закрытия 
   редактора проверить, был ли он изменён, и сообщить об этом Total Commander. Если вам не требуется 
   подобная опция, то вы можете вовсе отключить её (раздел Управление, переключатель Не следить за 
   запущенными приложениями).
